SNTP 原理



Originate Timestamp（发起时间）       T1     time request sent by client    
Receive Timestamp （接收时间）         T2     time request received at server    
Transmit Timestamp（发送时间）        T3     time reply sent by server    
Destination Timestamp（终点时间）     T4     time reply received at client 

t 为服务器和客户端之间的时间偏差；d 为两者之间的往返时间:
∵ T2=T1+t+d/2; T2-T1=t+d/2; T4=T3-t+d/2; T3-T4=t-d/2;
∴ d=(T4-T1)-(T3-T2); t=((T2-T1)+(T3-T4))/2;
求出了t和d，SNTP Client就可据此算出当前的时间。
如：now clock = T4+t。





PTP
      一般的同步技术是软件打时间戳，但是在发送消息时，消息CPU 处理(打时间戳) 到物理接口的时间是不确定的；如图3 所示，C到A的延时是不确定；相应地，接收消息也存在这个问题。这个等待时间可能会有几十个毫秒，严重影响了精度。PTP 提出同步消息在MAC 层和物理层之间打时间戳，即硬件打时间戳，图3 中，PTP 的时间戳处理在A 点，NTP 的时间戳处理在C 点，严重影响时间恢复的精度。硬件打时间戳延时抖动一般在数个纳秒之内，在离出入接口最近的地方打时间戳大大消除了协议栈等延迟的影响。
