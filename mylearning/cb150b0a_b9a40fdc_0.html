1. 首先，输入git status（查看本地分支文件信息，以查看在上次提交之后是否有修改，确保更新时不产生冲突）。

    比如：目前所在的分支是bugfix，显示了bugfix的上一次提交的文本信息

  2. 2


    我们需要将bugfix上的更新文件合并到dev上

    输入git branch ，可查看当前创建的分支名

    如图：有test，bugfix，dev，master

  3. 3

    选择你要合并到那个分支，先进行分支切换

    git checkout – [branchName]

    如图：切换到dev分支，git checkout dev

  4. 4

    远端更新

    如果是本地分支的话，则需要先切换到远端分支进行更新

    输入：git checkout remote branch

    如果已经是远端分支的话，就可以直接

    输入：git pull

    如果有文件在远端分支发生的改变，而本地还是之前没改变的分支节点，那么更新结果会打印出部分文件的改变信息


    git remote -v 

    可查看分支路径
